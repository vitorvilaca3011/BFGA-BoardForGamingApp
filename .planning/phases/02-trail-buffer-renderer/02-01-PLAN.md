    **GREEN phase:** Create `src/BFGA.Canvas/Rendering/LaserTrailBuffer.cs`:
    - `namespace BFGA.Canvas.Rendering;` (file-scoped)
    - `public sealed class LaserTrailBuffer` (not a record — mutable internal state)
    - Constructor: `LaserTrailBuffer(int capacity = 128)` — allocates `(Vector2 Position, long TimestampMs)[]` array of `capacity` size
    - Private fields: `_points` array, `_head` (int, next write index), `_count` (int)
    - `public int Count => _count;`
    - `public int Capacity => _points.Length;`
    - `public void Add(Vector2 position, long timestampMs)` — writes at `_head`, increments `_head % capacity`, increments `_count` up to capacity
    - `public ReadOnlySpan<(Vector2 Position, long TimestampMs)> GetPoints()` — returns valid points oldest-to-newest. When not wrapped: `_points[0.._count]`. When wrapped: must copy to temp array (oldest = `_head`, newest = `_head-1` wrapping). Use internal scratch array to avoid allocation.
    - Alternative simpler approach: `public void CopyPointsTo(Span<(Vector2, long)> destination, out int count)` or just return `(Vector2, long)[]` — pick whichever is simplest for the renderer to consume. The renderer iterates sequentially, so array return is fine.
    - `public void Clear()` — sets `_count = 0`, `_head = 0` (no need to zero array)
    - Per D3: timestamps use `Environment.TickCount64` (caller passes this in, buffer just stores it)
    - Zero allocation after construction: the backing array is allocated once, `Add` overwrites in-place

    Run tests — all 6 must pass.
