<objective>
Implement the local Laser Pointer gesture in `BoardView` so press, drag, release, quick tap, and cancellation all drive the new local overlay path without modifying board state.

Purpose: This is the user-facing phase. The toolbar already exposes the Laser Pointer tool, but the canvas input pipeline still routes everything through board-editing tools. This plan adds the local-only gesture path and keeps it bounded to Phase 3 scope.

Output: `BoardView` laser gesture helpers, cancellation hooks for exit/capture-loss/tool-switch, crosshair cursor mapping, and regression tests proving the gesture stays ephemeral.
</objective>

    ```csharp
    private void BeginLocalLaser(Vector2 boardPoint, Point screenPoint, long timestampMs)
    private void UpdateLocalLaser(Vector2 boardPoint, long timestampMs)
    private void CompleteLocalLaser(Vector2 boardPoint, Point screenPoint, long timestampMs)
    private void CancelLocalLaser()
    ```

    Implement the helpers with these rules:
    1. `BeginLocalLaser`:
       - create `LocalLaser = new LocalLaserState(_boardScreenViewModel!.SelectedStrokeColor)` if needed
       - set `IsActive = true`, `HeadPosition = boardPoint`, `LastUpdateMs = timestampMs`
       - clear the trail, then add the starting point once
       - store press start time and press start screen position for quick-tap detection
       - set `LocalPing = null`

    2. `UpdateLocalLaser`:
       - if `LocalLaser` is active, set `HeadPosition = boardPoint`
       - append `boardPoint` to `Trail` only when it differs from the previous point
       - keep all stored positions in board coordinates from `viewport.ScreenToBoard(...)`

    3. `CompleteLocalLaser`:
       - compute elapsed milliseconds and screen movement from stored press-start values
       - if `elapsed < 200` and `movement < 5`, set `LocalPing = new PingMarkerState(boardPoint, timestampMs, LocalLaser.Color)`
       - set `LocalLaser.IsActive = false` and `LocalLaser.LastUpdateMs = timestampMs`
       - do not clear the trail buffer; final fade-out must remain visible

    4. `CancelLocalLaser`:
       - if `LocalLaser` exists, set `IsActive = false` and update `LastUpdateMs = Environment.TickCount64`
       - do not create a ping and do not clear the trail

    5. Short-circuit pointer handlers before `TryHandlePointer(...)` when `CurrentTool == BoardToolType.LaserPointer`:
       - `HandlePointerPressed` calls `BeginLocalLaser(...)`, captures pointer, handles event, returns
       - `HandlePointerMoved` calls `UpdateLocalLaser(...)` while captured, handles event, returns
       - `HandlePointerReleased` calls `CompleteLocalLaser(...)`, releases capture, handles event, returns

    6. Add `viewport.PointerCaptureLost += HandlePointerCaptureLost;` in the constructor and implement:
       ```csharp
       private void HandlePointerCaptureLost(object? sender, PointerCaptureLostEventArgs e)
       {
           CancelLocalLaser();
       }
       ```

    7. Update `HandlePointerExited` so if the current tool is `LaserPointer` and the pointer is captured by `viewport`, it calls `CancelLocalLaser()`.

    8. Update `HandleBoardScreenPropertyChanged` selected-tool branch so switching away from `BoardToolType.LaserPointer` calls `CancelLocalLaser()` before `UpdateCursorForTool()`.

    9. Update `src/BFGA.App/Infrastructure/BoardCursorFactory.cs` to map `BoardToolType.LaserPointer => new Cursor(StandardCursorType.Cross)`.

    10. Do not modify `MainWindow.TryHandleToolShortcut` or assign a new keyboard shortcut for Laser Pointer in this phase.
